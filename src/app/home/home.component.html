Prueba Técnica para Frontend

Presentada por: Joel Guerrero

Para la prueba se utilizó Angular en su versión 15.0.4, aplicando pruebas unitarias utilizando Karma, para la emulación del Backend se utilizó la herramienta web mockapi.io, esta última permitió generar la estructura solicitada, además de generando respuestas de acuerdo con el enunciado.


Análisis del Problema

Se analiza la estructura de las peticiones con sus respectivos response, teniendo en cuenta que el problema parte de que se tiene un backend con respuesta de Formularios ya existente, se genera una api mockup y se crear datos fakes, con un rango medio de aleatoriedad para tener márgenes menores de errores. La api que se creó se en encuentra en el servidor con dirección

https://63b2fe5b5901da0ab3742ed6.mockapi.io/, de ahí se configura los endpoints de back:
  • GET: https://63b2fe5b5901da0ab3742ed6.mockapi.io/form/:id
  • POST: https://63b2fe5b5901da0ab3742ed6.mockapi.io/form/:id
    •

    Desarrollo

    Se crean dos componentes uno denominado Home que tendrá la funcionalidad inicial, pero no tiene mucha lógica, solo es para servir esta información. EL otro se denomina CurrentComponent y tiene toda la lógica para controlar los formularios tanto entrantes como los salientes, se apoya en dos servicios, uno que se llama formServices y el otro que se llama HttpServices, el primero tiene las función de convertir y gestionar el transporte desde y para el componente, así como la conversión de las respuesta recibidas del servidor, y todo ese proceso lo realiza utilizando HttpServices , que se encarga de gestionar los endpoints y de proveer la ruta de forma literal.

    Para renderizar los formularios se utilizó la renderizaron dinámica de angular, utilizando FormBuilder, esto permitió que la construcción de los campos sea un poco más sencillo, de esta forma la utilidad Validators de Angular tendría mas sentidos para especificar directamente los campos requeridos.

    Las rutas creadas para la gestión del front son:

    • http://localhost:4200/Home
    • http://localhost:4200/form/:id

    Se crean dos modelos uno que se llama FormModel y Properties, los dos son para gestionar los tipos de datos en el sistema, sin embargo fue mas utili crear desde el FormBuilder con los campos entrantes y con las propiedades.


    Retrospectiva

    Se intento realizar la prueba en su totalidad, sin embargo, luego de aplicar unas pruebas unitarias resultaron un 75% total pasadas, los errores visto son conexión “No provider for HttpClient!” lo que me hace pensar que falla en conexión al servidor o en la demora de conexión y en funcionamiento es cuando se tiene respuesta 400 http. Otro error que no se cumplió es si el servidor responde con un error diferente al mostrado en la prueba, por ejemplo, si el servidor backend esta offline, puede tener una salida no esperada. Considero que la parte visual puede mejorar y la funcionalidad en algunos detalles muy específicos se pueden optimizar de forma que sea mucho más reactiva. La herramienta Mockup Fake no tiene salida POST por medio de un id, y es difícil gestionar o configurar, por lo que las pruebas se hicieron por un método put, sin embargo se puede cambiar al método POST el cual fue probado en Local.
